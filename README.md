## Overview

The repository is organized into two parts.

- `code_l1/`: sparse optimization with an infinity-norm data-fitting term and an L1 regularizer
- `code_lowrank/`: low-rank matrix recovery with nuclear-norm regularization

The implementation is written in MATLAB and includes both custom algorithms and test scripts for reproducing the experiments.

## Part I: Sparse Regression with L1 Regularization

The first problem solved in this repository is

```math
\min_{x \in \mathbb{R}^n} \; \Vert A x - b \Vert_\infty + \mu \Vert x \Vert_1 .
```

Here:

- $A \in \mathbb{R}^{m \times n}$ is the data matrix,
- $b \in \mathbb{R}^m$ is the observation vector,
- $\mu > 0$ is the regularization parameter.

### Split Formulation

Introduce an auxiliary variable $z = A x - b$. Then the problem can be rewritten as

```math
\min_{x,z} \; \Vert z \Vert_\infty + \mu \Vert x \Vert_1
```

subject to

```math
A x - z - b = 0 .
```

This is the form used to derive the ALM / ADMM-type algorithms.

### Augmented Lagrangian

A standard augmented Lagrangian for the split problem is

```math
\mathcal{L}_\rho(x,z,u)
=
\mu \Vert x \Vert_1
+
\Vert z \Vert_\infty
+
u^\top (A x - z - b)
+
\frac{\rho}{2} \Vert A x - z - b \Vert_2^2 .
```

### Soft-Thresholding Step

The $x$-subproblem involves the proximal operator of the L1 norm. Its componentwise form is

```math
\mathrm{shrink}(v,\tau)_i
=
\mathrm{sign}(v_i)\max(|v_i|-\tau,0) .
```

In `L1ADMM.m`, the $x$-update is implemented as a proximal-gradient step:

```math
x^{k+1}
=
\mathrm{shrink}
\Big(
x^k - \frac{0.1}{\rho_k} A^\top (A x^k - z^k - b + u^k/\rho_k),
\frac{0.1 \mu_k}{\rho_k}
\Big) .
```

### Subgradient Step for the $z$-Update

The code uses a subgradient-type update for the $z$-subproblem. Define

```math
g^k = \rho_k(-A x^{k+1} + b + z^k) - u^k .
```

Let $I$ be an index satisfying

```math
|z_I^k| = \max_i |z_i^k| .
```

Then a subgradient of the infinity norm is chosen by

```math
h_i^k =
\begin{cases}
\mathrm{sign}(z_I^k) / \rho_k, & i = I, \\
0, & i \ne I .
\end{cases}
```

The update direction is then

```math
z_{\mathrm{new}}^{k+1}
=
-\alpha_k
\frac{g^k + h^k}{\Vert g^k + h^k \Vert_2} .
```

After that, a relaxation step is applied:

```math
z^{k+1}
=
\alpha \, z_{\mathrm{new}}^{k+1}
+
(1-\alpha) z^k .
```

### Dual Update

The dual variable is updated by

```math
u^{k+1}
=
u^k + 0.1 \rho_k (A x^{k+1} - z^{k+1} - b) .
```

### Test Data and Evaluation

In `testL1all.m`, the synthetic data are generated by

```math
A \in \mathbb{R}^{m \times n}, \qquad
u_{\star} \text{ sparse}, \qquad
b = A u_{\star} .
```

The script compares the custom methods against CVX-based baselines.

The reported quantities include:

```math
\mathrm{res}(x) = \Vert A x - b \Vert_1 ,
```

```math
\mathrm{nrm1}(x) = \Vert x \Vert_1 ,
```

and the relative error to the CVX solution

```math
\mathrm{err}(x_{\mathrm{cvx}}, x_{\mathrm{alg}})
=
\frac{\Vert x_{\mathrm{cvx}} - x_{\mathrm{alg}} \Vert_2}
{1 + \Vert x_{\mathrm{cvx}} \Vert_2} .
```

## Part II: Low-Rank Matrix Recovery

The second part of the repository studies low-rank matrix recovery from partially observed entries.

Let $\Omega$ be the set of observed indices. The sampling operator $P_\Omega$ is defined by

```math
[P_\Omega(X)]_{ij}
=
\begin{cases}
X_{ij}, & (i,j)\in\Omega, \\
0, & (i,j)\notin\Omega .
\end{cases}
```

The optimization problem implemented in the code is

```math
\min_{X \in \mathbb{R}^{m \times n}}
\;
\mu \Vert X \Vert_*
+
\Vert P_\Omega(X-M) \Vert_F^2 ,
```

where:

- $\Vert X \Vert_* = \sum_i \sigma_i(X)$ is the nuclear norm,
- $\Vert Y \Vert_F^2 = \sum_{i,j} Y_{ij}^2$ is the squared Frobenius norm.

### Smooth Part and Gradient

Define the smooth term

```math
f(X) = \Vert P_\Omega(X-M) \Vert_F^2 .
```

Its gradient is

```math
\nabla f(X) = 2 P_\Omega(X-M) .
```

### Accelerated Proximal-Gradient Method

The proximal-gradient method in `lowrank_prox.m` uses an extrapolation step of the form

```math
Y^k
=
X^k
+
\frac{k-2}{k+1}(X^k - X^{k-1}) .
```

Then a gradient step is performed:

```math
Z^k = Y^k - t_k \nabla f(Y^k) .
```

Equivalently,

```math
Z^k = Y^k - 2 t_k P_\Omega(Y^k - M) .
```

If the singular value decomposition of $Z^k$ is

```math
Z^k = U \Sigma V^\top ,
```

then singular value thresholding gives

```math
X^{k+1}
=
U \, \max(\Sigma - \mu t_k, 0) \, V^\top .
```

In scalar form, if $\Sigma = \mathrm{diag}(\sigma_1,\dots,\sigma_r)$, then

```math
X^{k+1}
=
U \, \mathrm{diag}\big((\sigma_1-\mu t_k)_+, \dots, (\sigma_r-\mu t_k)_+\big) \, V^\top ,
```

where

```math
(a)_+ = \max(a,0) .
```

### ADMM Split Formulation

The ADMM implementation is based on the split problem

```math
\min_{X,E}
\;
\mu \Vert X \Vert_*
+
\Vert P_\Omega(E) \Vert_F^2
```

subject to

```math
X + E = M .
```

This is equivalent to writing

```math
\min_{X,E}
\;
\mu \Vert X \Vert_*
+
\sum_{(i,j)\in\Omega} E_{ij}^2
```

subject to

```math
X + E = M .
```

### ADMM Updates

For the $X$-update, define

```math
S^k = M - E^k - \beta_k \Lambda^k .
```

If

```math
S^k = U \Sigma V^\top ,
```

then the update is

```math
X^{k+1}
=
U \, \max(\Sigma - \mu \beta_k, 0) \, V^\top .
```

For the $E$-update, define

```math
T^k = M - X^{k+1} - \beta_k \Lambda^k .
```

Then the update is entrywise:

```math
E_{ij}^{k+1}
=
\begin{cases}
\dfrac{T_{ij}^k}{2\beta_k+1}, & (i,j)\in\Omega, \\
T_{ij}^k, & (i,j)\notin\Omega .
\end{cases}
```

The dual variable is updated by

```math
\Lambda^{k+1}
=
\Lambda^k
+
\frac{\tau}{\beta_k}(X^{k+1}+E^{k+1}-M) .
```

### Data Generation in `Test_all.m`

The test script generates a low-rank ground-truth matrix in factored form:

```math
A = X_L X_R^\top ,
```

where $X_L \in \mathbb{R}^{m \times r}$ and $X_R \in \mathbb{R}^{n \times r}$.

The observed matrix is then constructed by keeping only the sampled entries:

```math
M_{ij}
=
\begin{cases}
A_{ij}, & (i,j)\in\Omega, \\
0, & (i,j)\notin\Omega .
\end{cases}
```
